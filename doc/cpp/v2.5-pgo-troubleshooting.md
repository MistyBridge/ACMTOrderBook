# v2.5 PGO 问题排查指南

## 常见问题

### 问题 1：Profile 数据合并失败

**现象**：
- 编译警告：没有合并 profile 数据
- 链接器警告：No profile data found
- 性能提升不明显

**原因**：
- MSVC PGO 的 .pgc 文件默认生成在可执行文件同目录
- 不是在 CMake 指定的 profile 目录

**解决步骤**：

1. **检查 .pgc 文件位置**
```bash
# 检查可执行文件目录
dir build_pgo\Release\*.pgc

# 检查 profile 目录
dir pgo_profiles\*.pgc
```

2. **手动复制 .pgc 文件**
```bash
copy build_pgo\Release\*.pgc pgo_profiles\
```

3. **使用脚本自动处理**
```bash
scripts\build_pgo.bat all
```

---

### 问题 2：性能提升不明显

**现象**：
- PGO 版本性能与基线版本持平
- 甚至略有下降

**可能原因**：

1. **Profile 数据质量差**
   - 未使用真实数据收集
   - 代码路径覆盖不全
   - 收集次数不足

2. **Profile 数据未正确合并**
   - 链接器未使用 profile 数据
   - .pgc 文件位置错误

3. **代码特点不适合 PGO**
   - 分支预测已经很好
   - 热函数已经内联
   - 代码路径单一

**解决步骤**：

1. **检查链接器是否使用了 profile 数据**
```bash
# 编译时查看链接器输出
cmake --build build_pgo --config Release --verbose 2>&1 | findstr "profile"
```

2. **增加 profile 数据收集次数**
```bash
# 多次运行收集更多数据
build_pgo\Release\orderbook_v2.exe "../data/20220422/AX_sbe_szse_000001.log" 0 2 16384 64 10
build_pgo\Release\orderbook_v2.exe "../data/20220422/AX_sbe_szse_000001.log" 0 2 16384 64 10
build_pgo\Release\orderbook_v2.exe "../data/20220422/AX_sbe_szse_000001.log" 0 2 16384 64 10
```

3. **检查 .pgc 文件是否生成**
```bash
dir build_pgo\Release\*.pgc
```

---

### 问题 3：编译失败

**现象**：
- CMake 配置失败
- 编译错误

**解决步骤**：

1. **检查 CMake 版本**
```bash
cmake --version
```

2. **检查编译器版本**
```bash
cl.exe --version
```

3. **清理构建目录**
```bash
rmdir /s /q build_pgo
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=GEN
```

---

### 问题 4：程序运行失败

**现象**：
- 程序崩溃
- 无输出

**解决步骤**：

1. **检查数据文件**
```bash
dir "../data/20220422/AX_sbe_szse_000001.log"
```

2. **使用调试版本**
```bash
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=GEN -DCMAKE_BUILD_TYPE=Debug
cmake --build build_pgo --config Debug
```

3. **检查 .pgc 文件是否生成**
```bash
dir build_pgo\Debug\*.pgc
```

## 调试技巧

### 1. 查看编译输出

```bash
cmake --build build_pgo --config Release --verbose
```

### 2. 查看链接器输出

```bash
cmake --build build_pgo --config Release --verbose 2>&1 | findstr "profile"
```

### 3. 检查 profile 数据

```bash
# MSVC
dir build_pgo\Release\*.pgc

# GCC
find build_pgo -name "*.gcda"
```

### 4. 验证 PGO 是否生效

```bash
# 编译时查看是否有 /USEPROFILE
cmake --build build_pgo --config Release --verbose 2>&1 | findstr "USEPROFILE"
```

## 最佳实践

### 1. Profile 数据收集

- 使用真实生产数据
- 覆盖所有代码路径
- 多次运行取平均值
- 包含边界情况

### 2. 编译配置

- 保持与生产环境一致
- 使用相同的优化选项
- 启用 LTO/LTCG

### 3. 性能测试

- 多次运行取平均值
- 统计标准差
- 超过 2× stddev 才算有效
- 100 次重放验证稳定性
