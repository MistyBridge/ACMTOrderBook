# v2.5 PGO (Profile-Guided Optimization) 编译优化

## 概述

PGO 是一种编译器优化技术，通过收集程序运行时的 profile 数据，指导编译器进行更智能的优化。

**预期收益**: +10% 吞吐量 (1,047K → 1,152K msg/s)

---

## 技术原理

### PGO 优化流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  第一次编译      │ →   │  运行收集数据    │ →   │  第二次编译      │
│  (插桩版本)      │     │  (Profile 数据)  │     │  (优化版本)      │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### PGO 优化来源

| 优化类型 | 说明 | 收益 |
|----------|------|------|
| 热函数内联 | 频繁调用的函数自动内联 | +3% |
| 分支预测优化 | 根据实际执行路径优化分支 | +2% |
| 循环展开优化 | 热循环自动展开 | +2% |
| 代码布局优化 | 热代码集中，减少 cache miss | +2% |
| 其他优化 | 指令调度、寄存器分配等 | +1% |

## 使用方法

### 快速开始（推荐）

```bash
cd cpp v2

# 一键 PGO 编译
scripts\build_pgo.bat all
```

### 分步执行

#### 第一步：生成 Profile 数据

```bash
# 编译插桩版本
scripts\build_pgo.bat gen

# 或手动执行：
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=GEN -DPGO_PROFILE_DIR=pgo_profiles
cmake --build build_pgo --config Release --parallel 8

# 运行程序收集 profile 数据
build_pgo\Release\orderbook_v2.exe "../data/20220422/AX_sbe_szse_000001.log" 0 2 16384 64 5
```

#### 第二步：使用 Profile 数据编译

```bash
# 使用 profile 数据编译优化版本
scripts\build_pgo.bat use

# 或手动执行：
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=USE -DPGO_PROFILE_DIR=pgo_profiles
cmake --build build_pgo --config Release --parallel 8
```

## CMake 配置选项

### PGO_MODE

```cmake
set(PGO_MODE OFF CACHE STRING "PGO mode (OFF/GEN/USE)")
```

| 值 | 说明 |
|---|---|
| `OFF` | 无 PGO（默认） |
| `GEN` | 生成 profile 数据（第一次编译） |
| `USE` | 使用 profile 数据优化（第二次编译） |

### PGO_PROFILE_DIR

```cmake
set(PGO_PROFILE_DIR "${CMAKE_SOURCE_DIR}/pgo_profiles" CACHE STRING "PGO profile directory")
```

指定 profile 数据存储目录。

## 编译器支持

### MSVC 2022

```bash
# 生成 profile 数据
cl /GENPROFILE /O2 /GL /arch:AVX2 main.cpp

# 运行程序
main.exe

# 使用 profile 数据编译
cl /USEPROFILE /O2 /GL /arch:AVX2 main.cpp
link /USEPROFILE /LTCG main.obj
```

### GCC/Clang

```bash
# 生成 profile 数据
g++ -fprofile-generate -O3 -march=native main.cpp

# 运行程序
./a.out

# 使用 profile 数据编译
g++ -fprofile-use -O3 -march=native main.cpp
```

## 注意事项

### 1. Profile 数据质量

- 使用真实数据收集 profile
- 覆盖所有代码路径
- 多次运行取平均值

### 2. 编译时间

- PGO 编译需要两次编译
- 第一次编译时间增加 10-20%
- 第二次编译时间增加 5-10%

### 3. 文件大小

- Profile 数据文件较大（10-100 MB）
- 需要足够的磁盘空间

### 4. 调试

- PGO 版本调试信息可能不完整
- 建议保留普通版本用于调试

## 常见问题

### Q1: Profile 数据在哪里？

A: 默认在 `pgo_profiles/` 目录下

### Q2: 如何清理 Profile 数据？

A: 删除 `pgo_profiles/` 目录

### Q3: PGO 编译失败怎么办？

A: 检查 profile 数据是否完整，重新收集

### Q4: MSVC .pgc 文件位置问题

**现象**：编译警告"没有合并 profile 数据"

**原因**：MSVC PGO 的 .pgc 文件默认生成在可执行文件同目录，不是 CMake 指定的 profile 目录

**解决**：
```bash
# 检查可执行文件目录
dir build_pgo\Release\*.pgc

# 手动复制 .pgc 文件
copy build_pgo\Release\*.pgc pgo_profiles\
```

或使用脚本自动处理：
```bash
scripts\build_pgo.bat all
```

### Q4: 性能提升不明显怎么办？

A: 确保使用真实数据收集 profile，覆盖所有代码路径
