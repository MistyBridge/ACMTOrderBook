# v2.5 PGO 编译工作流

## 快速开始

### 一键 PGO 编译（推荐）

```bash
cd cpp v2

# 完整 PGO 流程
scripts\build_pgo.bat all

# 性能对比测试
scripts\benchmark_pgo.bat
```

---

## 详细步骤

### 步骤 1：生成 Profile 数据

```bash
# 编译插桩版本
scripts\build_pgo.bat gen
```

**执行内容**：
1. 清理旧的 profile 数据
2. 配置 CMake（PGO_MODE=GEN）
3. 编译插桩版本
4. 运行程序收集 profile 数据

**输出文件**：
- `build_pgo/Release/orderbook_v2.exe` - 插桩版本
- `pgo_profiles/` - Profile 数据目录

---

### 步骤 2：使用 Profile 数据编译

```bash
# 使用 profile 数据编译优化版本
scripts\build_pgo.bat use
```

**执行内容**：
1. 配置 CMake（PGO_MODE=USE）
2. 使用 profile 数据编译优化版本

**输出文件**：
- `build_pgo/Release/orderbook_v2.exe` - PGO 优化版本

---

### 步骤 3：性能对比测试

```bash
# 运行性能对比
scripts\benchmark_pgo.bat
```

**执行内容**：
1. 运行基线版本 10 次
2. 运行 PGO 版本 10 次
3. 计算平均吞吐量
4. 计算性能提升百分比

**输出结果**：
```
============================================================
 性能对比结果
============================================================

 基线版本: 1047234 msg/s
 PGO 版本: 1152456 msg/s

 性能提升: 10%
============================================================
```

## CMake 配置

### PGO 模式选项

```cmake
# 生成 profile 数据
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=GEN

# 使用 profile 数据
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=USE

# 禁用 PGO
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=OFF
```

### Profile 数据目录

```cmake
# 默认目录
set(PGO_PROFILE_DIR "${CMAKE_SOURCE_DIR}/pgo_profiles")

# 自定义目录
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_PROFILE_DIR=/path/to/profiles
```

## 编译器选项

### MSVC 2022

**生成 Profile 数据**：
```bash
cl /GENPROFILE /O2 /GL /arch:AVX2 /utf-8 main.cpp
link /GENPROFILE /LTCG main.obj
```

**使用 Profile 数据**：
```bash
cl /USEPROFILE /O2 /GL /arch:AVX2 /utf-8 main.cpp
link /USEPROFILE /LTCG main.obj
```

### GCC/Clang

**生成 Profile 数据**：
```bash
g++ -fprofile-generate=./pgo_profiles -O3 -march=native main.cpp
```

**使用 Profile 数据**：
```bash
g++ -fprofile-use=./pgo_profiles -O3 -march=native main.cpp
```

## 故障排除

### 问题 1：CMake 配置失败

**错误**：
```
CMake Error: PGO_MODE must be OFF, GEN, or USE
```

**解决**：
```bash
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=GEN
```

### 问题 2：Profile 数据为空

**错误**：
```
PGO: No profile data found
```

**解决**：
1. 确保运行了插桩版本
2. 检查 `pgo_profiles/` 目录
3. 确保程序正常退出

### 问题 3：编译时间过长

**原因**：
- PGO 需要两次编译
- Profile 数据分析需要时间

**解决**：
- 使用并行编译：`--parallel 8`
- 减少 profile 数据收集次数

### 问题 4：性能提升不明显

**原因**：
- Profile 数据质量差
- 代码路径覆盖不全

**解决**：
1. 使用真实数据收集 profile
2. 增加 profile 数据收集次数
3. 确保覆盖所有代码路径

## 最佳实践

### 1. Profile 数据收集

- 使用真实生产数据
- 覆盖所有代码路径
- 多次运行取平均值
- 包含边界情况

### 2. 编译配置

- 保持与生产环境一致
- 使用相同的优化选项
- 启用 LTO/LTCG

### 3. 性能测试

- 多次运行取平均值
- 统计标准差
- 超过 2× stddev 才算有效
- 100 次重放验证稳定性

### 4. 版本管理

- 保留普通版本用于调试
- 记录 PGO 版本性能
- 定期更新 profile 数据
