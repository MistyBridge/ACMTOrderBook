## 2. Issues (Ranked by Impact)

### [RED] Issue 1: Consumer uses SwitchToThread()

```cpp
// consumer.cpp:56-65
if (n == 0) {
    emptyCount++;
    if (emptyCount < 10) { threadYield(); } // 1-10us!
    else { threadYield(); emptyCount = 0; } // same!
    continue;
}
```

**Problem**: SwitchToThread() = 1-10us per call (OS context switch)
emptyCount logic does nothing (both branches identical)

**Fix**:
```cpp
if (n == 0) {
    for (int i = 0; i < 32; ++i) cpu_pause(); // ~320ns
    continue;
}
```
Savings: **~200-400ns** per idle cycle

### [RED] Issue 2: Producer Per-Message Push

```cpp
while (!queue.try_push(ev)) { // 1 msg at a time
    for (int i = 0; i < 64; ++i) cpu_pause();
    std::atomic_thread_fence(acquire); // REDUNDANT
}
```

**Problem**: 1 atomic store/msg (~30ns cross-core), no batching
Consumer uses pop_batch but producer does NOT use push_batch

**Fix**: Producer-side buffering
```cpp
static constexpr int PB = 64;
MarketEvent pbuf[PB]; int bi = 0;
pbuf[bi++] = ev;
if (bi >= PB) {
    size_t pushed = 0;
    while (pushed < bi) {
        size_t n = queue.push_batch(pbuf+pushed, bi-pushed);
        pushed += n;
        if (n==0) for(int i=0;i<32;++i) cpu_pause();
    }
    bi = 0;
}
```
Savings: **~25-35ns/msg** (1 atomic per 64 msgs)

### [YELLOW] Issue 3: Batch Buffer 128KB (exceeds L1)

```cpp
static constexpr size_t MAX_BATCH = 1024; // 1024*128B = 128KB!
MarketEvent batch[MAX_BATCH];
```
Problem: 128KB > L1 cache (32-48KB), L1 misses on early items
Configured batchSize=64 completely IGNORED

Fix: MAX_BATCH = 64 (~8KB, fits L1)
Savings: **~10-30ns/msg**

### [YELLOW] Issue 4: Non-Trivially-Copyable MarketEvent

Custom copy ctor forces SPSC to use placement new instead of memcpy.
pop_batch: element-by-element move+destroy instead of bulk memcpy.

Fix: Remove custom copy/move ctors (all union members are POD)
Savings: **~15-30ns** per queue op

### [YELLOW] Issue 5: Per-Message Latency Measurement

```cpp
uint64_t dequeueTime = now_ns(); // ~30ns
latency.record(dequeueTime - batch[i].enqueueTimestamp); // ~10ns
```
Total: ~80ns/msg (2x now_ns + record)

Fix: Sample 1-in-16
```cpp
if ((totalConsumed & 0xF) == 0) { ... }
```
Savings: **~40-100ns/msg**
