## 完成工作

### 1. 依赖管理配置

**文件**: `cpp v2/CMakeLists.txt`

添加了 FetchContent 支持，可自动下载和集成三个候选哈希表库：

```cmake
option(USE_ROBIN_HOOD "Use robin_hood::unordered_map" OFF)
option(USE_PHMAP "Use phmap::flat_hash_map (Google abseil)" OFF)
option(USE_UNORDERED_DENSE "Use ankerl::unordered_dense::map" OFF)
option(BUILD_BENCHMARK "Build hashmap benchmark" ON)
```

**支持的库**:

| 库 | Git Tag | 集成方式 | 编译选项 |
|---|---|---|---|
| robin_hood | 3.11.5 | FetchContent | `-DUSE_ROBIN_HOOD=ON` |
| phmap (abseil) | 20230802.0 | FetchContent | `-DUSE_PHMAP=ON` |
| ankerl::unordered_dense | v2.0.0 | FetchContent | `-DUSE_UNORDERED_DENSE=ON` |

### 2. 基准测试框架

**文件**: `cpp v2/benchmark/hashmap_benchmark.cpp`

测试场景:
1. **Sequential Insert**: 顺序插入 500,000 个元素
2. **Random Find**: 随机查找 1,000,000 次
3. **Random Erase**: 随机删除 500,000 个元素
4. **Mixed Ops**: 混合负载 (插入 30% / 查找 50% / 删除 20%)
5. **Memory Est.**: 内存占用估算

特性:
- 支持条件编译测试各哈希表库
- 固定随机种子，结果可复现
- 自动预热和多次迭代取平均

### 3. 一键运行脚本

**文件**: `cpp v2/benchmark/run_benchmark.bat`

用法:
```bash
.\benchmark\run_benchmark.bat std              # 仅测试 std::unordered_map
.\benchmark\run_benchmark.bat robin_hood       # 测试 robin_hood
.\benchmark\run_benchmark.bat phmap            # 测试 phmap::flat_hash_map
.\benchmark\run_benchmark.bat all              # 测试所有哈希表
```

## 技术细节

### CMake FetchContent 配置

```cmake
# robin_hood (header-only)
FetchContent_Declare(
    robin_hood
    GIT_REPOSITORY https://github.com/martinus/robin-hood-hashing.git
    GIT_TAG        3.11.5
)
FetchContent_MakeAvailable(robin_hood)
target_compile_definitions(orderbook_v2 PRIVATE USE_ROBIN_HOOD)
target_link_libraries(orderbook_v2 PRIVATE robin_hood::robin_hood)

# phmap (abseil - header-only subset)
FetchContent_Declare(
    abseil
    GIT_REPOSITORY https://github.com/abseil/abseil-cpp.git
    GIT_TAG        20230802.0
)
set(ABSL_CXX_STANDARD 17)
FetchContent_MakeAvailable(abseil)
target_compile_definitions(orderbook_v2 PRIVATE USE_PHMAP)
target_link_libraries(orderbook_v2 PRIVATE absl::flat_hash_map)

# ankerl::unordered_dense (header-only)
FetchContent_Declare(
    unordered_dense
    GIT_REPOSITORY https://github.com/martinus/unordered_dense.git
    GIT_TAG        v2.0.0
)
FetchContent_MakeAvailable(unordered_dense)
target_compile_definitions(orderbook_v2 PRIVATE USE_UNORDERED_DENSE)
target_link_libraries(orderbook_v2 PRIVATE unordered_dense::unordered_dense)
```

### 基准测试编译选项

```cmake
if(BUILD_BENCHMARK)
    add_executable(hashmap_benchmark benchmark/hashmap_benchmark.cpp)
    target_compile_features(hashmap_benchmark PRIVATE cxx_std_17)

    # MSVC 特定选项
    if(MSVC)
        target_compile_options(hashmap_benchmark PRIVATE
            /utf-8 /O2 /arch:AVX2
        )
    else()
        target_compile_options(hashmap_benchmark PRIVATE
            -O3 -march=native
        )
    endif()

    # 链接哈希表库（如果启用）
    if(USE_ROBIN_HOOD)
        target_link_libraries(hashmap_benchmark PRIVATE robin_hood::robin_hood)
        target_compile_definitions(hashmap_benchmark PRIVATE USE_ROBIN_HOOD)
    endif()
    # ... phmap 和 unordered_dense 类似
endif()
```

## 使用示例

### 1. 运行基准测试

```bash
cd cpp v2

# 测试所有哈希表
.\benchmark\run_benchmark.bat all
```

### 2. 手动构建

```bash
cd cpp v2

# 生成构建系统（启用所有哈希表）
cmake -B build -G "Visual Studio 17 2022" -A x64 \
  -DBUILD_BENCHMARK=ON \
  -DUSE_ROBIN_HOOD=ON \
  -DUSE_PHMAP=ON \
  -DUSE_UNORDERED_DENSE=ON

# 编译
cmake --build build --config Release --target hashmap_benchmark --parallel 8

# 运行
build\Release\hashmap_benchmark.exe
```

### 3. 集成到主程序

选择最优哈希表后，修改 CMakeLists.txt：

```cmake
# 启用选中的哈希表
option(USE_PHMAP "Use phmap::flat_hash_map" ON)
```

然后修改 `behave/axob.h`：

```cpp
#ifdef USE_PHMAP
#include "absl/container/flat_hash_map.h"
phmap::flat_hash_map<uint64_t, ObOrder*> orderMap;
#else
std::unordered_map<uint64_t, ObOrder*> orderMap;
#endif
```

## 注意事项

1. **首次编译**: FetchContent 首次编译会下载依赖，需要网络连接
2. **编译时间**: abseil 库较大，首次编译时间较长（约 2-3 分钟）
3. **内存占用**: abseil 编译时内存占用较高，建议 8GB+ 内存
4. **MSVC 兼容性**: 所有库均支持 MSVC 2022 (v143)
