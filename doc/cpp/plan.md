# C++ 撮合引擎重写计划

## 目标
将 Python py/ 目录下的订单簿重建引擎重写为 C++，保持算法完全一致，同时做模块化拆分。

---

## 版本历史

| 版本 | 状态 | 说明 |
|------|------|------|
| CPP v1 | ✅ 已完成 | 单线程版本，基础 C++ 重写 |
| CPP v2 | ✅ 已完成 | 多线程优化版本（生产者-消费者架构） |
| CPP v2.1 | ✅ 已完成 | MSVC 2022 编译优化 + HybridLevelBook |

## CPP v2：多线程优化版本

### v2 核心优化
| 优化项 | 说明 | 状态 |
|--------|------|------|
| SPSC 无锁队列 | 环形缓冲区 + atomic 读写指针 | ✅ 已完成 |
| 内存池 | 侵入式空闲链表，2x 增长 | ✅ 已完成 |
| 缓存行对齐 | alignas(64) 防止 false sharing | ✅ 已完成 |
| 批量处理 | pop_batch 一次取 64 条 | ✅ 已完成 |
| CPU 亲和性 | 生产者/消费者绑定不同物理核 | ✅ 已完成 |
| 延迟统计 | 环形缓冲区 + nth_element | ✅ 已完成 |
| 字段重排 | ObOrder 40B → 32B | ✅ 已完成 |
| 分支预测 | LIKELY/UNLIKELY 宏 | ✅ 已完成 |
| 编译优化 | -O2 -march=native | ✅ 已完成 |

### 性能指标（GCC -O2）
- 吞吐量：95,455 msg/s（+47.7% vs v1）
- 延迟：p50=0.3μs, p99=71.5μs, p99.9=521.8μs, pmax=957.3μs
- 编译选项：-O2 -march=native

## CPP v2.1：MSVC 编译优化 + HybridLevelBook

### 完成日期
2026-06-14

### 核心优化

| 优化项 | 说明 | 状态 |
|--------|------|------|
| MSVC 2022 编译 | /O2 /GL /arch:AVX2 /LTCG | ✅ 已完成 |
| HybridLevelBook | 排序数组(n≤256) + std::map 回退 | ✅ 已完成 |
| 分支预测宏 | LIKELY/UNLIKELY 热路径优化 | ✅ 已完成 |
| 字段重排 | ObOrder 40B → 32B，消除 padding | ✅ 已完成 |

### 性能指标（MSVC 2022）
- 吞吐量：**212,817 msg/s**（+123% vs v2 GCC，+198% vs v1）
- 10次重放稳定：214,644 msg/s
- 延迟：p50=0.3μs, p99=71.5μs
- 编译选项：MSVC 2022 /O2 /GL /arch:AVX2 /LTCG

### 构建方式
```bash
# 方式1：CMake（推荐）
cmake -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release --parallel 8

# 方式2：MSBuild（快速）
MSBuild.exe build/orderbook_v2.vcxproj //p:Configuration=Release //p:Platform=x64 //m:8
```
