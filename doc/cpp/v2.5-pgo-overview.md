## PGO 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    PGO 编译流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 编译插桩版本 (PGO_MODE=GEN)                            │
│     └→ 生成带 profiling 的可执行文件                       │
│                                                             │
│  2. 运行程序收集 Profile 数据                              │
│     └→ 生成 pgo_profiles/*.gcda 或 *.pgc                   │
│                                                             │
│  3. 编译优化版本 (PGO_MODE=USE)                            │
│     └→ 使用 profile 数据优化代码生成                       │
│                                                             │
│  4. 性能测试验证                                           │
│     └→ 对比普通版本和 PGO 版本                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 编译命令参考

### CMake 配置

```bash
# 生成 profile 数据
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=GEN

# 使用 profile 数据
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=USE

# 禁用 PGO
cmake -B build_pgo -G "Visual Studio 17 2022" -A x64 -DPGO_MODE=OFF
```

### 编译

```bash
# 编译
cmake --build build_pgo --config Release --parallel 8
```

## 注意事项

### 1. Profile 数据质量

- 使用真实数据收集 profile
- 覆盖所有代码路径
- 多次运行取平均值

### 2. 编译时间

- PGO 编译需要两次编译
- 第一次编译时间增加 10-20%
- 第二次编译时间增加 5-10%

### 3. 文件大小

- Profile 数据文件较大（10-100 MB）
- 需要足够的磁盘空间

### 4. 调试

- PGO 版本调试信息可能不完整
- 建议保留普通版本用于调试

## 常见问题

### Q1: Profile 数据在哪里？

A: 默认在 `pgo_profiles/` 目录下

### Q2: 如何清理 Profile 数据？

A: 删除 `pgo_profiles/` 目录

### Q3: PGO 编译失败怎么办？

A: 检查 profile 数据是否完整，重新收集

### Q4: 性能提升不明显怎么办？

A: 确保使用真实数据收集 profile，覆盖所有代码路径

### Q5: 如何禁用 PGO？

A: 使用 `-DPGO_MODE=OFF` 或不设置 PGO_MODE
