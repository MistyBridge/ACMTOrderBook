# v2.6 零分配解析 + 二分查找 + ankerl hashmap 报告

## 📊 优化概述

**优化目标**：消除堆分配开销，优化数据结构性能。

**优化方法**：
1. 零分配 advance()：std::string → 栈缓冲区 + parseI64
2. 二分查找 find()：HybridLevelBook O(n) → O(log n)
3. ankerl hashmap：orderMap 切换到 ankerl::unordered_dense
4. 延迟采样优化：now_ns() 每 8 条采样一次

**优化效果**：吞吐量从 1,052K msg/s 提升至 1,125K msg/s，**提升 7%**。

---

## 🔧 技术实现

### 1. 零分配 advance()

**优化前**：
```cpp
std::string line(lineStart, ptr_);  // 堆分配
order.loadFromLine(line.c_str());
```

**优化后**：
```cpp
// 直接用指针对，零分配
order.loadFromLine(lineStart, lineEnd);
```

**效果**：
- 消除每条消息的堆分配
- 减少内存碎片
- 提升缓存命中率

### 2. 二分查找 find()

**优化前**：O(n) 线性扫描
**优化后**：O(log n) 二分查找

**效果**：
- 查找时间从 O(n) 降至 O(log n)
- 适合大规模数据

### 3. ankerl hashmap

**优化前**：std::unordered_map（链表法）
**优化后**：ankerl::unordered_dense（开放寻址）

**效果**：
- 缓存友好
- 减少内存碎片
- 提升查找性能

### 4. 延迟采样优化

**优化前**：每条消息调用 now_ns()
**优化后**：每 8 条消息采样一次

**效果**：
- 减少 QPC 调用开销
- 节省 ~13ns/msg
